## 1. 产品定位

**Project Pilot** 是一个本地运行的 GitHub 开源项目探索管理工具，帮助用户在 **发现 → 收集 → 整理 → 理解 →（可选）内容创作** 的流程中保持秩序。

核心价值：**让每一个感兴趣的项目都有迹可循、找得到、读得懂、推得出去。**

> 2026-06-27 起**不再**以「辅助本地部署、看板体验流转」为主轴；见 [§0 方向修订](#0-方向修订2026-06-27)。

## 2. 核心痛点

| # | 痛点描述 |
|---|---|
| P1 | GitHub 信息收集方式原始，靠收藏链接或截图，信息碎片化 |
| P2 | 项目部署方式多种，混用后忘记怎么启动 | **不再单独做部署日志/辅助**；理解依赖 README + 外链工具 |
| P3 | 项目靠本地文件夹分类管理，多领域兴趣下分类混乱、难以统一查找 |
| P4 | 缺乏快速体验的入口，项目一多就懒得去试，积累大量「未体验」项目 |

## 3. 功能矩阵

### 3.1 收集（Import）

**入口：**
- 粘贴 GitHub 仓库 URL
- （未来扩展）浏览器书签栏快捷按钮

**收集流程：**
1. 用户粘贴 GitHub 仓库 URL
2. 后端自动抓取仓库基础信息（项目名、描述、Star 数、语言、作者、许可证）
3. AI 读取 README 文件，生成项目摘要（核心能力 + 适用场景）
4. AI 解析部署方式，生成部署标签

**部署标签类型：**
```
Docker / npm / pip / pipx / 二进制下载 / 源码编译 / 在线Demo / 云部署按钮 / GitHub Codespaces
```

📋 未体验 ──→ ⚡ 正在体验 ──→ ✅ 推荐归档 ──+──→ ❌ 放弃归档
                              |            |
                          推荐项目        放弃项目
                        （内部分区）    （内部分区）

**状态规则：**
- 新项目默认进入「未体验」

- 归档后内部分区（推荐 / 放弃），归档时间自动记录

### 3.3 领域标签（Tags）

每个项目可打多个领域标签，支持自定义：

```
#AI  #DevTools  #自动化  #数据分析  #效率工具  #开源平台  #...
```

看板支持按标签筛选，不依赖本地文件夹路径。

### 3.4 项目详情（Detail）

**目标能力**（产品全貌）：

| 区块 | 内容 |
|---|---|
| 基础信息 | 名称、描述、Star、语言、作者、许可证、GitHub 地址 |
| AI 摘要 | README 解析结果，1-3 段话总结核心能力和适用场景 |
| 部署标签 | AI 识别的部署方式列表 |
| 部署日志 | 记录每次启动的时间、方式、备注（手动 / 自动生成启动命令） |
| 个人笔记 | 自由编辑区，支持 Markdown，体验结论、推荐理由、放弃原因等 |
| 快速入口 | 跳转到 GitHub / Demo / 文档 |

**当前 Web 实现**（`/projects/:id`，2026-05-17 起，详见 [`CHANGELOG_2026-05-17.md`](../changelogs/CHANGELOG_2026-05-17.md)）：

| 区块 | 实现说明 |
|---|---|
| 布局 | **Steam 式**全宽英雄区 + 底部分割 **Tab 栏**（README / Release / 笔记）；顶栏 **返回资料库**（与资料库主区一致） |
| 英雄区 | GitHub 头像、项目名、体验状态、`owner/repo` **外链**；统计行 **Stars / Forks / 推送时间**（进入页自动 `refresh-github?scope=stats`）；简介 **双击编辑**、标题旁 **翻译**（Sparkles，译文覆盖 `description`）；领域标签行悬停 **「+」** |
| Tab · README | `GET /projects/{id}/readme` 应用内 **Markdown**；**分段机器翻译**（`GET .../readme/blocks`、`POST .../translate/readme-block`），译文存 `readme_translated`；**右键** 原文/译文/编辑/重试失败段（见 [`CHANGELOG_2026-05-24.md`](../changelogs/CHANGELOG_2026-05-24.md)） |
| Tab · Release | `GET /projects/{id}/releases`，**卡片列表**（圆角边框）；**标题** 链至 GitHub Release；`tag_name` 在标题右侧（与标题相同时不重复） |
| Tab · 笔记 | `projects.notes` 字段，`PATCH /projects/{id}` 显式保存（Ctrl+S） |
| 更多信息 | Tab 栏右侧 **Popover**：AI 摘要、部署方式、**许可证**、语言、文件夹、收录/更新时间、最新 Release 等（英雄区不再展示许可证与独立 GitHub 按钮） |
| 待办 | Demo 入口（可选）；~~部署日志~~、~~看板~~ 已移出方向 |

## 4. 数据存储

### 4.1 主数据库

**SQLite**，包含以下核心表：

- `projects` — 项目基础信息、AI 摘要、状态、标签、添加时间；**`deleted_at`** 非空表示在回收站（软删除）
- `deploy_logs` — 部署日志
- `notes` — 个人笔记
- `tag_categories` — 用户自定义标签分类（名称唯一、排序）
- `tags` — 标签定义（可选 `category_id`；为空表示**未分类**）
- `project_tags` — 项目-标签关联

**实现说明（与 OpenAPI 一致）**：`projects` 含 **`deleted_at`**（可空）：**`DELETE /projects/{id}`** 为软删（移入回收站），**`GET /projects?deleted_only=true`** 列回收站，**`POST .../restore`** 恢复，**`DELETE .../permanent`** 物理删除；资料库树与默认列表不含已删项。`tags` 含 `name`（唯一）、`category_id`（外键至 `tag_categories`，**ON DELETE SET NULL**，删除分类后标签回到未分类）。`project_tags` 为 `(project_id, tag_id)` 多对多关联，删除项目或标签时级联清理关联行。标签与分类在 **资料库 → 标签管理** 中维护（含拖拽归类；前端「标签分类」为 `DragOverlay`、分类栏碰撞、`PATCH` 乐观更新、**无**松手退回动画，见 [`changelogs/CHANGELOG_2026-05-13.md`](../changelogs/CHANGELOG_2026-05-13.md)）；**项目绑定**：`PATCH /projects/{id}` 请求体可含 `tag_ids`（整数数组，整表替换关联）；列表侧 **`GET /projects?missing_tags=true`** 用于「无任一标签」筛选；`tags.usage_count` 由关联行计数驱动，与上述闭环一致。

### 4.2 Obsidian 笔记同步

每个项目自动生成一条 Obsidian Markdown 笔记，路径可配置：
```
.github-projects/<项目名>.md
```

笔记结构：
```markdown
# <项目名>

## 基础信息
- GitHub: <url>
- Stars: ⭐ <count>
- 语言: <language>

## AI 摘要
<AI 生成的项目描述>

## 部署方式
- Docker
- npm

## 部署日志
<最近一次部署记录>

## 笔记
<个人笔记内容>
```

- 单向同步（Project Pilot → Obsidian），不反向回写
- 用户可选择开启或关闭此功能

## 5. 技术栈

| 层级 | 技术选型 |
|---|---|
| 后端 | FastAPI + SQLite（aiosqlite） |
| 前端 | React + Refine + shadcn/ui |
| AI 摘要 | 支持配置 OpenAI API / 本地模型接口 |
| 部署 | 本地运行，可通过反向代理局域网访问 |
| 数据 | SQLite 本地文件存储 |

### 5.1 资料库主界面（Web，已实现对齐说明）

与当前前端实现一致（详见根目录 [`README.md`](../README.md)）：

- **整体**：语雀式 **窄条功能区** → **库侧栏**（约 `w-72`，树与快捷入口）→ **主内容**（`/library` 下板块与表格/卡片）→ **可拖拽右栏**（`react-resizable-panels` 的 `feature-drawer`，与主区分隔条可调宽；顶栏可 **收起 / 展开**）。
- **右栏**：在资料库且 **非**「标签管理」、**非**回收站时显示。**无**选中项目卡片时，按当前板块展示 **摘要**（名称、数量等）或 **文件夹信息**（名称可编辑等）；**有**选中时展示 **项目预览**（单击预览、双击进入项目详情路由）：预览 **顶栏**可整块进入详情（路径链 GitHub 不冒泡）；简介 **双击**编辑、**失焦**保存、**Escape** 放弃；领域标签行悬停 **「+」** 管理标签。右栏已收起时，单击卡片会先 **展开** 再预览。侧栏切换文件夹或板块时会 **结束项目预览**，以便右栏回到摘要或文件夹信息。
- **主区项目展示**：顶栏可切换 **网格**（整齐等高）与 **瀑布流**（多列错落、高度随卡片内容）；偏好可本地持久化。
- **主内容滚动**：`/library` 主区 `<main>` 滚动条默认弱化隐藏，**滚动时**短暂显示以便拖动。
- **GitHub 项目卡片**：支持 **右键菜单**（复制仓库链接、打开站内详情、移动到文件夹、**移入回收站** 确认），与拖拽归类并存；**回收站**内为 **恢复** / **彻底删除**。
- **实现注意**：`react-resizable-panels` v4 中面板尺寸的 **百分比** 须写成带单位的字符串（如 `"68%"`），勿与 **像素** 裸数字混淆。

## 7. 非功能需求

- **隐私优先**：所有数据存在本地，不上传云端
- **轻量化**：SQLite 无需独立数据库服务，开箱即用
- **快速响应**：AI 摘要异步生成，不阻塞添加流程
- **无感归档**：状态切换不强制要求填写内容，不增加操作负担
