# Project Pilot — 桌面安装包打包经验

> 文档版本：v1.1  
> 更新日期：2026-06-01  
> 适用：**Windows · Tauri 2 + FastAPI sidecar（PyInstaller）+ NSIS 安装包**  
> 关联：[桌面工程化指南](./PROJECT_PILOT_Desktop_Engineering_Guide.md)、[发版检查清单](./PROJECT_PILOT_Release_Checklist.md)、[README 桌面章节](../README.md#桌面tauri--sidecarwindows)

本文记录 **Phase 0 首次打出可安装桌面包** 过程中的环境准备、构建命令、验证步骤与踩坑修复，供后续发版与排障参照。架构设计与 Phase 路线图仍以 [桌面工程化指南](./PROJECT_PILOT_Desktop_Engineering_Guide.md) 为准。

## 一、产物与架构速览

| 组件 | 技术 | 说明 |
|------|------|------|
| 窗口壳 | Tauri 2（Rust） | `project-pilot.exe`，单实例、启停 sidecar、WebView |
| 后端 | FastAPI sidecar | `project-pilot-api.exe`（PyInstaller onefile，体积约 50MB+） |
| 前端 | React 静态资源 | `frontend/dist` 打包进安装目录 |
| 安装包 | NSIS | `Project Pilot_0.1.2_x64-setup.exe` |
| 用户数据 | SQLite | `%LOCALAPPDATA%\ProjectPilot\` |

**生产态请求路径**（方案 A）：

```text
WebView → http://127.0.0.1:38472/
              ├── /          → SPA 静态页
              ├── /api/...   → FastAPI REST
              └── /health    → sidecar 就绪探测
```

与 **网页开发态** 对比：

| 维度 | 网页版（开发） | 桌面版（安装包） |
|------|----------------|------------------|
| 前端 | Vite `:5173` | sidecar 同源提供 `dist/` |
| 后端 | uvicorn `:8000` | sidecar 固定 `:38472` |
| 数据库 | `backend/project_pilot.db` | `%LOCALAPPDATA%\ProjectPilot\database\project_pilot.db` |

三者端口 **互不占用**，网页版与桌面版理论上可同时运行；数据与 Token **不共享**（见 [七、网页版与桌面版并存](#七网页版与桌面版并存)）。

## 二、环境准备

除 [README](../README.md) 中的 Python 3.11+、Node.js 20+ 外，桌面打包还需：

| 依赖 | 用途 | 安装 |
|------|------|------|
| Rust stable (MSVC) | 编译 Tauri | [rustup.rs](https://rustup.rs/) → `stable-x86_64-pc-windows-msvc` |
| WebView2 | Tauri WebView | Win10/11 通常已预装 |
| PyInstaller | 打 sidecar exe | `pip install -e "backend/.[dev,desktop]"` |

**首次构建** Rust 会下载 crate 并编译较久（数分钟～十余分钟属正常）；sidecar 首次 PyInstaller 亦需等待。

## 三、标准构建流程

**必须在仓库根目录执行**（Tauri 配置路径、`beforeBuildCommand`、sidecar 复制路径均相对根目录）。

### 3.1 一键完整构建

```powershell
# 仓库根目录
.\scripts\build-desktop.ps1
```

等价于：

1. `scripts/build-sidecar.ps1` — 前端 build → 复制到 `src-tauri/resources/dist` → PyInstaller → 复制到 `src-tauri/binaries/project-pilot-api-x86_64-pc-windows-msvc.exe`
2. `npx --prefix frontend tauri build --config src-tauri/tauri.conf.json`

### 3.2 分步构建（后端或 sidecar 变更后）

```powershell
.\scripts\build-sidecar.ps1    # 仅 sidecar + 静态资源
npx --prefix frontend tauri build --config src-tauri/tauri.conf.json
```

### 3.3 产物位置

```text
src-tauri\target\release\project-pilot.exe          # 可直接运行验证
src-tauri\target\release\bundle\nsis\
    Project Pilot_0.1.2_x64-setup.exe               # 分发给用户的安装包
```

安装包 **勿提交 git**（见 [`.gitignore`](../.gitignore)）；发版上传 [GitHub Releases](./PROJECT_PILOT_Release_Checklist.md)。

## 四、关键配置要点

### 4.1 Tauri 命令必须在仓库根目录

在 `frontend/` 或 `src-tauri/` 内单独跑 `tauri build` 容易路径错乱。统一：

```powershell
npx --prefix frontend tauri build --config src-tauri/tauri.conf.json
```

### 4.2 静态资源映射

[`src-tauri/tauri.conf.json`](../src-tauri/tauri.conf.json) 使用 **map 形式** 将构建时复制的 dist 打进安装包：

```json
"resources": {
  "resources/dist/": "dist/"
}
```

Rust 启动时按多个候选路径解析 `index.html`（`dist`、`resources/dist` 等），避免安装后找不到前端文件。

**重要**：NSIS 打进安装包的是 **`src-tauri/resources/dist`**（见 `bundle.resources`），不是 `frontend/dist`。`tauri build` 的 `beforeBuildCommand` 与 [`scripts/sync-frontend-to-tauri-resources.ps1`](../scripts/sync-frontend-to-tauri-resources.ps1) 会在 build 后 **同步** `frontend/dist → resources/dist`。若只跑 `npm run build` 或只更新 `frontend/dist` 而不同步，安装包会继续携带**旧前端**（侧栏可能出现 `Not Found`、缺少项目库切换 UI 等）。

### 4.3 Sidecar 命名与 externalBin

PyInstaller 产出 `project-pilot-api.exe`，复制为 Tauri 要求的 target triple 后缀名：

```text
src-tauri/binaries/project-pilot-api-x86_64-pc-windows-msvc.exe
```

`tauri.conf.json` 中 `"externalBin": ["binaries/project-pilot-api"]`。

### 4.4 Sidecar 环境变量（Rust 注入）

| 变量 | 桌面生产值 | 说明 |
|------|------------|------|
| `HOST` | `127.0.0.1` | 仅本机 |
| `PORT` | `38472` | 固定端口 |
| `DATABASE_URL` | AppData 下绝对路径 | 与开发库分离 |
| `STATIC_DIR` | 安装目录内 `dist` 绝对路径 | 非空时 FastAPI 挂载 SPA |
| `CORS_ORIGINS` | **JSON 数组字符串** | 见 [五.1](#51-安装后无窗口cors_origins-格式) |
| `LOG_DIR` | `%LOCALAPPDATA%\ProjectPilot\logs` | sidecar 日志 |

### 4.5 安装包格式：NSIS 优先

首次 `tauri build` 若同时打 MSI（WiX），可能因 **下载 WiX 工具链超时** 失败。当前 [`tauri.conf.json`](../src-tauri/tauri.conf.json) 设为 `"targets": ["nsis"]` 即可稳定产出 `.exe` 安装程序。

## 五、踩坑与解决方案

### 5.1 安装后无窗口（CORS_ORIGINS 格式）

**现象**：安装成功，任务管理器有 `project-pilot.exe` / `project-pilot-api.exe`，但 **没有任何界面**。

**根因**：Rust 传入 `CORS_ORIGINS=http://127.0.0.1:38472`；后端 [`config.py`](../backend/app/core/config.py) 中 `cors_origins: list[str]` 经 pydantic-settings 从环境变量解析时 **要求 JSON 数组**。解析抛 `JSONDecodeError` → sidecar 秒退 → Tauri health 轮询超时 → `setup` 失败 → 无窗口。

**修复**：[`src-tauri/src/lib.rs`](../src-tauri/src/lib.rs) 改为：

```rust
let cors_json = format!(r#"["http://127.0.0.1:{DESKTOP_PORT}"]"#);
.env("CORS_ORIGINS", cors_json)
```

**验证**：`%LOCALAPPDATA%\ProjectPilot\logs\desktop.log` 应出现：

```text
release: starting sidecar
sidecar spawned, waiting for /health
sidecar healthy
main window created
```

浏览器访问 `http://127.0.0.1:38472/health` 应返回 `{"status":"ok","version":"0.1.x"}`。

### 5.2 首次 `cargo check` / `tauri build` 失败

| 报错 | 原因 | 处理 |
|------|------|------|
| sidecar exe 不存在 | 未先跑 `build-sidecar.ps1` | 先构建 sidecar |
| resources/dist 为空 | 未复制前端 dist | 跑 sidecar 脚本或完整 desktop 脚本 |
| `PermissionDenied` 复制 sidecar | 旧 `project-pilot-api.exe` 仍被占用 | 任务管理器结束 `project-pilot*` 后重试 |

### 5.3 PyInstaller 冷启动慢 → health 超时

onefile sidecar **首次解压** 可能需数十秒。health 超时由 30s 调整为 **120s**；仍失败则查 sidecar 是否崩溃（见日志）。

### 5.4 启动失败无提示

修复前 sidecar 崩溃时用户无任何反馈。已增加：

- 启动日志：`%LOCALAPPDATA%\ProjectPilot\logs\desktop.log`
- 失败时 **`rfd` 错误对话框**（[`Cargo.toml`](../src-tauri/Cargo.toml) 依赖 `rfd`）

### 5.5 安装包前端仍是旧版（resources/dist 未同步）

**现象**：已 bump 版本并重装，但 UI 缺新功能；侧栏文件夹树红色 **Not Found**（旧 JS 仍请求已删除的 `/api/library/tree`），主区列表却可能有数据（legacy `/api/projects` 仍可用）。

**根因**：安装目录内 `dist/` 来自 **`resources/dist` 的旧副本**，与当前 `frontend/dist` 不一致。

**修复**（已纳入仓库脚本）：

```powershell
# 仓库根目录 — 统一 build + 同步 + sidecar + NSIS
.\scripts\build-desktop.ps1

# 或仅同步（已有 frontend/dist 时）
.\scripts\sync-frontend-to-tauri-resources.ps1 -SkipBuild
```

**验证**：

- 设置页或账户菜单 footer 显示 `Project Pilot v0.1.x (build 时间)`
- `http://127.0.0.1:38472/health` 返回 `"version":"0.1.x"`
- 在 `src-tauri/resources/dist/assets/*.js` 中搜索 `project-libraries` 应存在

### 5.7 打包后 UI 仍是旧版（WebView2 缓存 + dist 残留）

**现象**：`.\scripts\build-desktop.ps1` 已成功、`resources/dist` 与 `index.html` 已是新 hash（如 `index-C9q5ZnRj.js`），但 Tauri 窗口仍像旧版：无账户菜单版本号、资料库侧栏无「← 返回 / 项目库切换」、或侧栏 **Not Found**。用 Chrome 打开 `http://127.0.0.1:38472/` 的 Network 面板，可能仍加载 `index-ti9LZwoN.js` 等旧文件。

**根因（两层叠加）**：

1. **dist 目录残留旧 JS**  
   Tauri 将 `resources/dist` 增量复制到 `target/release/dist`（及安装目录 `dist/`）时，**不会删除**上次 build 留下的 `assets/index-*.js`。磁盘上同时存在多个 hash 文件；`index.html` 虽已指向新文件，但排查时容易误判「bundle 里已有新代码」。

2. **WebView2 强缓存**  
   桌面壳用 WebView2 打开 `http://127.0.0.1:38472/`。WebView 会缓存 **`index.html`**（以及曾加载过的 JS/CSS）。多次本地打包迭代后，即使 sidecar 已 serve 新 `index.html`，窗口仍可能用**缓存里的旧 index**，继续引用旧 hash 的 JS → 界面停留在数月前的 UI。  
   **这与「前端 sync 是否执行」无关**：sync 可以正确，缓存仍让你看到旧界面。

**修复**（已纳入仓库，2026-06-01 20:40 起）：

| 措施 | 位置 |
|------|------|
| build 前删除 `src-tauri/target/release/dist` | [`sync-frontend-to-tauri-resources.ps1`](../scripts/sync-frontend-to-tauri-resources.ps1) |
| 按 `index.html` 引用校验主 JS，禁止 assets 内多个大体积 `index-*.js` | [`build-desktop.ps1`](../scripts/build-desktop.ps1) |
| `GET /` 返回 `index.html` 时带 `Cache-Control: no-store` | [`backend/app/main.py`](../backend/app/main.py) |
| 启动前清理残留 sidecar、校验 `/api/project-libraries` | [`src-tauri/src/lib.rs`](../src-tauri/src/lib.rs) |

**开发/发版时手动清 WebView 缓存**（完全退出 `project-pilot*` 后）：

```powershell
Get-Process -Name "project-pilot*" -ErrorAction SilentlyContinue | Stop-Process -Force
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\com.projectpilot.app\EBWebView" -ErrorAction SilentlyContinue
```

然后重新运行 `src-tauri\target\release\project-pilot.exe` 或重装最新 NSIS。

**验证是否加载新前端**：

```powershell
# 1. 看 index.html 指向哪个 JS
Select-String -Path "src-tauri\target\release\dist\index.html" -Pattern 'assets/index-'

# 2. assets 里应只有 1 个大体积 index-*.js（约 1.4MB）
Get-ChildItem "src-tauri\target\release\dist\assets\index-*.js" | Where-Object { $_.Length -gt 10000 }

# 3. 应用运行时，浏览器 F12 → Network 刷新 http://127.0.0.1:38472/
#    主脚本文件名应与 index.html 一致
```

**UI 对照**：新包在资料库侧栏顶栏有 **← 返回** + **项目库名称下拉**；左下角头像 Popover 底部有 `Project Pilot v0.1.x (build 时间)`。

**以后如何避免**：

1. **只从仓库根目录**跑 `.\scripts\build-desktop.ps1`，不要只 `npm run build` 或单独 `tauri build`。
2. **测 release exe 前先** `Stop-Process project-pilot*`；发版前若 UI 可疑，清一次 EBWebView（见上）。
3. **不要混用** `target\release\project-pilot.exe` 与开始菜单旧安装；重装后确认 `Local\Project Pilot\project-pilot.exe` 修改时间与最新 setup 一致。
4. **怀疑 UI 版本时**：浏览器打开 `http://127.0.0.1:38472/` 看 Network 里实际 JS 文件名，比肉眼看窗口更可靠。

### 5.8 桌面版外链点击无反应

**现象**：打包后 GitHub 链接、README 内链、设置页文档链接等点击无反应；浏览器开发态（`npm run dev`）正常。

**根因**：

1. 生产态 WebView 加载 **`http://127.0.0.1:38472/`**（sidecar 同源），属于 Tauri 的 **remote URL**，默认不向该页面注入 `window.__TAURI__`。
2. 前端 `ExternalLink` 若走 `<a target="_blank">`，WebView2 **不会在应用内打开新标签**，表现为「点了没反应」。
3. 仅配置 `capabilities` 的 `remote.urls` 并调用 `@tauri-apps/plugin-opener` 的 `openUrl()` **仍可能静默失败**（IPC 在 sidecar origin 上不可用）；单靠 Rust `on_navigation` 若前端未触发 WebView 导航，同样不会执行。

**修复**（2026-06-01，第二轮已验证可用）：

| 层 | 改动 |
|----|------|
| Capabilities | [`default.json`](../src-tauri/capabilities/default.json) 增加 `remote.urls`（`127.0.0.1:38472`、`localhost:5173`） |
| 前端 | [`open-external-url.ts`](../frontend/src/lib/open-external-url.ts) — sidecar origin（`:38472`）上 **不依赖 IPC**，改为 `window.location.href = url` 触发 WebView 导航；dev / 有 `__TAURI__` 时仍优先 `openUrl()` |
| Rust | [`lib.rs`](../src-tauri/src/lib.rs) — `on_navigation` + `on_new_window` 拦截站外 `http(s)` / `mailto` / `tel`，用 `app.opener().open_url()` 打开系统浏览器，并写 `desktop.log` |
| 组件 | [`external-link.tsx`](../frontend/src/components/common/external-link.tsx) 统一走 `openExternalUrl()` |

**数据流**（sidecar 生产态）：

```text
用户点击外链
  → openExternalUrl() 设置 window.location.href = 外链
  → WebView 发起导航
  → Rust on_navigation 判定非 127.0.0.1:38472
  → 系统默认浏览器打开该 URL，WebView 留在应用内
```

**验证**：

1. 重新 `.\scripts\build-desktop.ps1` 后运行 `src-tauri\target\release\project-pilot.exe`（或安装最新 NSIS）。
2. 点击项目卡片 GitHub 链接、设置页 Token 文档链接 → 应弹出 **系统默认浏览器**。
3. `%LOCALAPPDATA%\ProjectPilot\logs\desktop.log` 应出现一行：`open external: https://...`。
4. 若仍无反应：确认 exe 修改时间与最新 build 一致；必要时清 EBWebView 缓存（见 [5.7](#57-打包后-ui-仍是旧版webview2-缓存--dist-残留)）。

**注意**：22:53 及更早的 0.1.2 build 可能只含第一轮修复（capabilities + `openUrl`），外链仍无效；须使用 **含导航拦截方案** 的 build（如主 JS `index-VEwby7kZ.js` 及之后）。

### 5.6 GitHub Token / 数据「丢失」

开发库 `backend/project_pilot.db` 与桌面库 `%LOCALAPPDATA%\ProjectPilot\database\project_pilot.db` **不是同一个文件**。在网页版配置的 Token、项目数据不会自动出现在桌面版；需在桌面应用 **设置 → GitHub** 重新保存，或自行迁移/导出。

GitHub 网站 **无法查看** 已创建 PAT 的明文，只能 Regenerate；本地 SQLite `app_settings` 表键 `github_personal_access_token` 可找回（若曾保存在对应库中）。

## 七、网页版与桌面版并存

**不是** `8000` / `5173` 与 `38472` 的端口冲突——端口本就分离。

常见「打不开」原因：

| 情况 | 说明 | 处理 |
|------|------|------|
| **单实例** | 已有 `project-pilot.exe` 在后台，再点图标只聚焦旧窗口 | 结束所有 `project-pilot*` 后重启 |
| **残留 sidecar** | 上次未正常退出，`project-pilot-api.exe` 仍占 `38472` | 同上 |
| **`tauri dev` + `npm run dev`** | 两者都需 **5173** | 二选一，不要同时开两个 Vite |

## 九、经验摘要

1. **契约先行**：API 统一 `/api` 前缀后，Vite 代理须 **透传**（去掉 rewrite），桌面 sidecar 与 Web 开发路径一致。
2. **list 型环境变量**：传给 pydantic-settings 时用 **JSON 数组**，不要用逗号分隔的普通字符串（除非 validator 明确支持）。
3. **构建目录**：Tauri / sidecar 脚本 **固定从仓库根目录** 执行。
4. **先 sidecar 后 bundle**：缺 exe 或缺 dist 时 `tauri build` 必失败。
5. **release exe 先于 setup 测**：省去反复安装卸载，确认 sidecar + 窗口 OK 再打 NSIS。
6. **数据目录分离**：桌面用户数据在 AppData，与 `backend/project_pilot.db` 无关；文档与测试时需说明。
7. **安装包不进仓库**：产物放 GitHub Releases；`.gitignore` 已忽略 `*.exe`、`target/` 等。
8. **dist 同步**：NSIS 打包的是 `resources/dist`；务必通过 [`sync-frontend-to-tauri-resources.ps1`](../scripts/sync-frontend-to-tauri-resources.ps1) 与 `tauri build` 的 `beforeBuildCommand` 保持与源码一致。
9. **WebView 缓存**：本地反复打包后，Tauri 窗口可能仍显示旧 UI；清 EBWebView 或依赖 sidecar 对 `index.html` 的 `no-store`（见 [5.7](#57-打包后-ui-仍是旧版webview2-缓存--dist-残留)）。
10. **dist 勿堆积**：`target/release/dist/assets` 不应并存多个大体积 `index-*.js`；build 脚本已会在 sync 时删除旧 `release/dist`。
11. **外链走导航拦截**：sidecar origin 上勿依赖 `window.__TAURI__` 或 `openUrl` IPC；前端触发 `location.href`，Rust `on_navigation` 用系统浏览器打开（见 [5.8](#58-桌面版外链点击无反应)）。
