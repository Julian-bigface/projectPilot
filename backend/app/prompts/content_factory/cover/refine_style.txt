## 硬性约束

1. 只输出一个合法 JSON 对象，不要 markdown 围栏，不要过程解释。
2. **prompt_template 正文须用中文**（七个占位符名保持 `{project_name}` 等英文蛇形不变）；必须保留全部占位符：`{project_name}`、`{project_description}`、`{headline}`、`{cover_texts}`、`{highlight_tags}`、`{project_language}`、`{project_stars}`；除上述七个外不要用花括号包裹其它词。
3. **prompt_prefix** 须含 `1242x1660, 3:4 vertical portrait cover, strict margins`（画幅段可保留；气质描述用中文为主）。
4. **design_analysis** 十维须与修订后的 prompt 字段一致；有实质解构内容时各字段应互相呼应。
5. **negative_prompt** 须具体，与 design_analysis / prompt 不冲突。
6. **color_tokens** / **font_tokens** 须与 design_analysis 的 color_strategy / typography_strategy 一致。
7. **style_report** 2–4 句中文：修订后为何适合开源推广、应避免什么。
8. 按用户意图修改；未提及的字段尽量保留原意，不要无必要推翻。
